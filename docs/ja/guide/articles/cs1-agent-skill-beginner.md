# v0.3.0 リリース内容: AI市長が街を読み、直し、保存できるようになりました

`cities-skylines1-agent-skill` v0.3.0 をリリースしました。

今回のリリースでは、Cities: Skylines 1の街づくりをAIエージェントに任せるための「見る」「直す」「確認する」「保存する」流れを強化しています。

AIがいきなり画面を見て適当にクリックするのではなく、ゲーム内の状態をAPIで読み取り、道路やゾーン、税率、問題アイコンを確認しながら、必要な操作を小さく実行できるようになりました。

## 今回のリリースで何ができるようになったの？

AIがCities: Skylines 1の中にいる「市長補佐」として動けるようになります。

できることは、たとえば次のような作業です。

- 街の人口、需要、問題アイコンを読む
- 道路、水道管、電線、施設の状態を調べる
- 道路がつながっているか、変な重なりがないか確認する
- 必要な道路やインフラを追加する
- 建物を置く、移動する、削除する
- ゾーンを塗る
- 税率やシミュレーション速度を変える
- セーブする

ポイントは、AIが「なんとなく画面を見てクリックする」のではなく、ゲームの中のデータを読んでから操作することです。

## どういう仕組み？

仕組みは大きく3つです。

1. Cities: Skylines 1用のMODを入れる
2. MODがローカルAPIを立ち上げる
3. CodexなどのAIエージェントがAPIを呼ぶ

APIは、AIがゲームにお願いを出すための窓口です。

たとえば、人間ならマウスで道路ツールを選んで道路を引きます。AIの場合は、次のようなお願いをAPIに送ります。

```text
この座標からこの座標まで、Basic Roadを作って
```

するとMODがCities: Skylines 1のゲーム内処理を呼び出して、実際に道路を作ります。

## なぜスクリーンショットだけではなくAPIなの？

AIにゲーム画面を見せるだけでも、ある程度は操作できそうに見えます。

でもCities: Skylines 1では、画面だけだと分かりにくいことが多いです。

- 道路が見た目では交差しているのに、ゲーム内部では接続していない
- 水道管や電線が本当に届いているか分かりにくい
- 問題アイコンが小さく、画面角度やズームで読み取りづらい
- セーブが本当に作られたか、画面だけでは確認しにくい

そこで、AIがゲーム内部のデータを直接読めるようにしています。

たとえば `/state/road-anomalies` では、道路の重なり、行き止まり、交差しているのにノードがない場所などを調べられます。`/state/saves` では、保存ファイルが実際に存在するか確認できます。

## v0.3.0の主なリリース内容

v0.3.0では、街を育てるための観察ポイントが増えました。

- 需要バーを読む `/state/demand`
- 市民メッセージを読む `/state/chirps`
- ゾーン面積を読む `/state/zones`
- 税率を読む `/state/economy`
- 外部道路とつながっているか調べる `/state/external-connections`
- ゾーンの混ざりや穴を調べる `/state/zone-anomalies`

操作側も増えています。

- 税率を変える `/commands/set-tax-rate`
- 建物をオン/オフする `/commands/set-building-active`
- 壊れたアセットを無効化する `/commands/disable-blocked-assets`
- ゾーンを既存建物に合わせて直す `/commands/repair-zones-to-growables`
- まだらなゾーンをまとめて直す `/commands/repair-zone-clusters`

つまり、AIが「街の状況を読む」「小さく直す」「もう一度確認する」という流れを作りやすくなりました。

## このリリースで使いやすくなった流れ

最初は、AIに全部任せるよりも、次のような小さな依頼から始めるのがおすすめです。

```text
この街の問題をAPIで確認して、道路やインフラの問題があれば1つずつ直して。
変更したら保存して、何をしたか報告して。
```

AI側は、だいたい次の流れで動きます。

1. `/state/summary` で街の概要を見る
2. `/state/problems` で問題アイコンを見る
3. `/state/road-anomalies` で道路の異常を見る
4. 必要なら `/commands/bulldoze` で悪い道路や建物を削除する
5. `/commands/build-network` や `/commands/place-building` で作り直す
6. `/commands/set-simulation-speed` で少し時間を進める
7. もう一度APIで確認する
8. `/commands/save` で保存する

大事なのは、一気に全部直さないことです。小さく直して、確認して、保存する。この繰り返しが安全です。

## このリリースはどんな人に向いている？

このプロジェクトは、次のような人に向いています。

- Cities: Skylines 1が好き
- AIエージェントにゲームを操作させてみたい
- 画面認識だけでなく、ゲーム内部データを使った自動化に興味がある
- MOD開発やローカルAPI連携を試してみたい
- AIに「考えて、実行して、確認して、保存する」流れを任せてみたい

逆に、まだ実験段階なので、大切な本番セーブでいきなり使うのはおすすめしません。まずはテスト用のセーブや新しいマップで試すのが安全です。

## まとめ

v0.3.0は、`cities-skylines1-agent-skill` を「AIがCities: Skylines 1を操作する実験」から、「AIが街の状態を読んで、根拠を持って直していく運用」に近づけるリリースです。

AIが街を操作するために大切なのは、派手な自動化よりも、次の基本です。

- 街の状態を正しく読む
- 小さな操作に分ける
- 操作後にもう一度確認する
- 保存して戻れるようにする

v0.3.0では、この基本ループを回しやすくするAPIと運用ツールが増えました。
